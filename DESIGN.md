## 📌 概要

Claude Assist は Claude Code の入力支援 GUI ツールです。
Windows Terminal + ConPTY 環境で動作し、Claude Code のチャット欄にテキストを送信します。

### プロセスモデル

#### 🔵 スタンドアロン起動（従来モード）

```
ターミナル ──→ uv run claude_assist.py
                │
                ├─ 1. HWND / Shell PID を取得
                ├─ 2. GUIプロセスを分離起動 (CREATE_NO_WINDOW)
                └─ 3. スクリプト終了
                      │
                      ▼
             GUIプロセス (--gui --hwnd N --pid N)
                │
                ├─ ✅ WriteConsoleInputW でテキスト送信
                └─ 🔄 ユーザーが閉じるまで常駐
```

#### 🟡 スキル起動（`/claude-assist`）

```
Claude Code ──→ Bash: uv run claude_assist.py --skill
                │
                ├─ 1. _find_terminal_window() で Windows Terminal HWND を発見
                ├─ 2. GUIプロセスを分離起動 (CREATE_NO_WINDOW)
                └─ 3. スクリプト即終了（Claude Code はブロックされない）
                      │
                      ▼
             GUIプロセス (--gui --hwnd N --pid 0)
                │
                ├─ ✅ Clipboard + Ctrl+V でテキスト送信
                └─ 🔄 ユーザーが閉じるまで常駐
```

## 📡 テキスト送信方式

### 方式1: `WriteConsoleInputW`（スタンドアロン時）

| 項目 | 内容 |
|------|------|
| 📍 使用場面 | スタンドアロン起動（PID 取得可能時） |
| ⚙️ API | Windows Console API |
| 📝 手順 | `FreeConsole()` → `AttachConsole(pid)` → `WriteConsoleInputW()` |
| ✅ 利点 | 高速・確実・クリップボード非使用 |
| ❌ 制約 | ConPTY 環境では使用不可 |

### 方式2: `Clipboard + Ctrl+V`（スキル時）

| 項目 | 内容 |
|------|------|
| 📍 使用場面 | スキル起動（ConPTY / Windows Terminal 環境） |
| ⚙️ API | `SetForegroundWindow()` + `keybd_event()` |
| 📝 手順 | クリップボードにセット → フォーカス切替 → `Ctrl+V` → `Enter` |
| ✅ 利点 | ConPTY 環境で動作 |
| ❌ 制約 | クリップボードを一時的に使用 |

### ⚡ スラッシュコマンドの特別処理

`/skill-name` で始まるテキストは、1文字ずつ送信するとオートコンプリートが
干渉するため、**方式に関わらずクリップボード貼り付けを使用**。

## 🔨 プロンプト構築ロジック

### `_build_prompt()` の出力形式

```
📌 ケース1: テキストのみ
   → "ユーザーの指示文"

📎 ケース2: テキスト + 添付ファイル/画像
   → "ユーザーの指示文 // @ファイルパス @画像パス"

📚 ケース3: テキスト + 参照情報
   → "ユーザーの指示文 // 以下の資料を熟読した上で指示を遂行せよ: @参照パス"

📎📚 ケース4: テキスト + 参照情報 + 添付ファイル
   → "ユーザーの指示文 // 以下の資料を熟読した上で指示を遂行せよ: @参照パス // @添付パス"
```

### 💡 設計意図

| ルール | 理由 |
|--------|------|
| 指示文を常に先頭に配置 | `/skill-name` が先頭にくる必要があるため |
| `//` 区切りで分離 | `\n` は WriteConsoleInputW でスペースに変換されるため使用不可 |
| 参照情報のみヒント文を付与 | 添付ファイル/画像には Claude への追加指示不要 |

## 🔍 ターミナルウィンドウ発見ロジック

### 🚧 課題

Windows Terminal は ConPTY を使用するため:
- `GetConsoleWindow()` → **0** を返す（コンソールウィンドウが存在しない）
- Windows Terminal の HWND はプロセスツリーの**親チェーンに含まれない**

### ✅ 解決策: `_find_terminal_window()`

全ウィンドウを `EnumWindows` で列挙し、スコアリングで最適なターミナルを選択:

| 条件 | スコア | 例 |
|------|:------:|-----|
| `WindowsTerminal.exe` のウィンドウ | **+50** | Windows Terminal |
| `cmd.exe` / `PowerShell` 等 | **+30** | コマンドプロンプト |
| タイトルに `"claude"` を含む | **+40** | `⠐ Claude Code` |
| タイトルに `"terminal"` `"cmd"` 等を含む | **+20** | `コマンド プロンプト` |

→ **最高スコアのウィンドウを選択**

## 💾 データ永続化

### ファイル構成

| ファイル | 形式 | 内容 | 上限 |
|----------|:----:|------|:----:|
| 📊 `data/history.json` | JSON 配列 | 入力履歴 | 50件 |
| 📊 `data/templates.json` | JSON 配列 | テンプレート `[{name, content}]` | 100件 |
| 📊 `data/config.json` | JSON オブジェクト | 言語設定、参照情報パス等 | — |

### 📂 データディレクトリの解決

```python
def _data_dir() -> Path:
    base = Path(__file__).resolve().parent
    if base.name == "scripts":
        base = base.parent  # → skills/claude-assist/
    return base / "data"
```

> `scripts/` 配下に配置 → 親ディレクトリ（スキルフォルダ）の `data/` を使用
> それ以外 → 同階層の `data/` を使用

## 🔒 セキュリティ考慮

| 対策 | 内容 |
|------|------|
| 📋 クリップボード | 送信時のみ使用、常駐的な監視はしない |
| 🖼️ 一時ファイル | `_paste_*.png` は次回貼り付け時・アプリ終了時に削除 |
| 📝 デバッグログ | `--debug` フラグ時のみ出力（通常時は無出力） |
| 📂 パス | 絶対パスのみ使用（パストラバーサル対策） |
